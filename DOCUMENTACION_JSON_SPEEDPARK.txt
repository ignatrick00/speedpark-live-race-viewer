================================================================================
                              CAMPOS CRÍTICOS PARA APP
================================================================================

🔑 IDENTIFICACIÓN ÚNICA:
- Campo "D": ID único permanente del corredor (ej: 32670842)
- Este ID NO cambia aunque el corredor use diferentes karts
- Perfecto para tracking histórico de mejores tiempos

⏱️ TIEMPOS PRINCIPALES:
- Campo "T": Tiempo de la última vuelta completada (más actual)
- Campo "B": Mejor tiempo personal de la sesión (récord personal)
- Campo "A": Tiempo acumulado o de sector intermedio

📊 MÉTRICAS DE CARRERA:
- Campo "P": Posición actual (1=primero, 2=segundo, etc.)
- Campo "L": Número de vuelta (incrementa con cada vuelta completada)
- Campo "G": Gap en segundos con el líder (formato "05.432")

🏁 IDENTIFICACIÓN DE SESIÓN:
- Campo "R": Número de heat/carrera dentro del día
- Campo "E": ID único de la sesión completa
- Campo "T" (raíz): Timestamp de inicio de la sesión

================================================================================
                          CONVERSIONES DE TIEMPO
================================================================================

FORMATO EN JSON: Milisegundos (ej: 43657)
CONVERSIÓN A SEGUNDOS: 43657 / 1000 = 43.657 segundos
FORMATO LEGIBLE: 0:43.657 (mm:ss.sss)

FUNCIÓN DE CONVERSIÓN:
function formatTime(milliseconds) {
    const totalSeconds = milliseconds / 1000;
    const minutes = Math.floor(totalSeconds / 60);
    const seconds = (totalSeconds % 60).toFixed(3);
    return `${minutes}:${seconds.padStart(6, '0')}`;
}

================================================================================
                               EVENTOS DETECTABLES
================================================================================

NUEVA VUELTA COMPLETADA:
- Detectar cuando campo "L" incrementa
- Capturar tiempo de esa vuelta desde campo "T"

NUEVO MEJOR TIEMPO:
- Detectar cuando campo "B" disminuye
- Registrar el nuevo récord personal

CAMBIO DE POSICIÓN:
- Detectar cuando campo "P" cambia
- Registrar movimiento en las posiciones

FINALIZACIÓN DE CARRERA:
- Detectar cuando "CE" cambia de 1 a 0
- Capturar posiciones finales y tiempos

================================================================================
                                NOTAS IMPORTANTES
================================================================================

1. PERSISTENCIA DE IDs:
   - El campo "D" (ID corredor) persiste entre sesiones
   - Usar como clave primaria para tracking histórico

2. TIEMPO MÁS CONFIABLE:
   - Campo "T" es el tiempo de última vuelta más preciso
   - Campo "B" es el mejor tiempo acumulado de la sesión

3. ESTADOS DE CARRERA:
   - "CE": 1 = carrera activa, 0 = carrera pausada/terminada
   - "CS": 1 = cronómetro corriendo, 0 = cronómetro parado

4. FORMATO DE GAP:
   - String vacío "" = es el líder
   - "04.271" = 4.271 segundos detrás del líder

5. FRECUENCIA DE DATOS:
   - Los datos llegan cada ~2-5 segundos
   - Se actualizan en tiempo real durante la carrera
